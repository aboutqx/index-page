Html5 new semantic elements

html5 定义了一些新的语义化的标签，包括<section>，<article> ,<nav>,<header>,<aside>,<hgroup>,<footer>,<time>,<mark>

如何兼容地使用这些标签

在ie9以前的ie浏览器上，新增的标签存在两个问题：

  1. ie8及以前的ie浏览器不会识别section等新标签，对于这些不识别的标签会创建一个没有子节点的空节点。所以html结构中的section的标签的子元素将会实际上被当作它的姐妹元素。
  2. ie及以前ie浏览器对于不识别的标签不会应用任何样式，section这些元素也不会被当作block-level element，而是应用不识别元素的初始样式inline。
像Paul Irish写的:the new elements cannot hold children and are unaffected by CSS.

使用html5 shiv来解决： 


<!--[if lt IE 9]>
<script>
// html5shiv (lite) | @afarkas @jdalton @jon_neal @rem | MIT/GPL2

// shiv elements on the document
"abbr article aside audio bdi canvas data datalist details figcaption figure footer header hgroup mark meter nav output progress section subline summary time video".replace(/\w+/g, function (nodeName) {
  document.createElement(nodeName);
});

// prepend html5 styles to the document
(function (element) {
  element.innerHTML = "x<style>article,aside,details,figcaption,figure,footer,header,hgroup,nav,section,subline,summary{display:block}audio,canvas,video{display:inline-block}audio:not([controls]){display:none;height:0}[hidden]{display:none}</style>";

  document.documentElement.insertBefore(element.lastChild, document.documentElement.firstChild);
})(document.createElement("x"));
</script>
<![endif]-->

html5 shiv 还包括monkeypatches for document.createElement and document.createDocumentFragment for IE6-8,解决了元素的innerHTML不识别新元素的问题，详见github:html5shiv。
注意：html5 shiv需要应用在页头，这样才确保起作用。

新标签的semantics

<section>
section 代表了一片普通文档或者程序的区块。 一个section是内容的主题部分，通常包含一个heading。section的例子有章节，tabbed 对话框的各个内容，或者论文的带数标的块。 一个网站的主页能被分成介绍，新闻条目，联系信息这些sections。
<nav>
nav 代表了页面链接到其他页面或者当前页面的其他部分的区块：包含导航链接的区块。 不是所有链接组合需要被放在 nav 元素里 — 仅仅包含主要导航块的适合 nav . 特别地, 页尾会包含一个网站的链接列表, 里面有服务条款，主页，版权页，footer 元素是足够的，而不需要nav .
<article>
article element represents a component of a page that consists of a self-contained composition in a document, page, application, or site and that is intended to be independently distributable or reusable, e.g. in syndication. This could be a forum post, a magazine or newspaper article, a Web log entry, a user-submitted comment, an interactive widget or gadget, or any other independent item of content.
<aside>
The aside element represents a section of a page that consists of content that is tangentially related to the content around the aside element, and which could be considered separate from that content. Such sections are often represented as sidebars in printed typography. The element can be used for typographical effects like pull quotes or sidebars, for advertising, for groups of nav elements, and for other content that is considered separate from the main content of the page.
<hgroup>
The hgroup element represents the heading of a section. The element is used to group a set of h1–h6 elements when the heading has multiple levels, such as subheadings, alternative titles, or taglines.
<header>
The header element represents a group of introductory or navigational aids. A header element is intended to usually contain the section’s heading (an h1–h6 element or an hgroup element), but this is not required. The header element can also be used to wrap a section’s table of contents, a search form, or any relevant logos.
<footer>
The footer element represents a footer for its nearest ancestor sectioning content or sectioning root element. A footer typically contains information about its section such as who wrote it, links to related documents, copyright data, and the like. Footers don’t necessarily have to appear at the end of a section, though they usually do. When the footer element contains entire sections, they represent appendices, indexes, long colophons, verbose license agreements, and other such content.
<time>
time element represents either a time on a 24 hour clock, or a precise date in the proleptic Gregorian calendar, optionally with a time and a time-zone offset.
<mark>
mark 元素代表了一份文档中被标记或高亮来作为引用的一系列文字。
